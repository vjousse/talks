Elle souligne l'application des fonctions, contrairement au modèle de programmation impérative qui met en avant les changements d'état. »

Et pour de vrai ?

C'est une façon d'écrire des programmes en n'utilisant que des fonctions pures.

Qu'est-ce qu'une fonction pure ?

C'est une fonction sans effet de bord (side effect).

Qu'est-ce qu'une fonction avec effet de bord ?

C'est une fonction qui fait autre chose que simplement retourner un résultat.

Mais concrêtement ?

En fait, chaque fonction dispose de deux ensembles d'entrées/sorties : vous n'en êtes certainement pas conscient pour l'instant.

Prenons un exemple :


public int square(int x) {
    return x * x;
}

=> Une entrée (les paramètres), une sortie (le return).

Et dans cet exemple, quelles sont les entrées/sorties ?


public void processNext() {
    Message message = InboxQueue.popMessage();

    if (message != null) {
        process(message);
    }
}

=> Pas de paramètres, pas de return.

On parle ici d'effets de bord (« side-effects »)

Effet de bord = Complexité


public void processMessage() {...}
  • Aucune idée de ce que fait cette fonction !
  • Code très difficile à comprendre/tester : il faut connaître le monde qui entoure la fonction.

Comment s'en débarrasser ?

En déterrant tous les effets de bord !

Un petit exemple


public Program getCurrentProgram(TVGuide guide, int channel) {
  Schedule schedule = guide.getSchedule(channel);

  Program current = schedule.programAt(new Date());

  return current;
}

Êtes-vous capable de trouver l'effet de bord ?

De quoi dépend cette fonction qui n'est pas spécifié en paramètre ?


public Program getProgramAt(TVGuide guide, int channel, Date when) {
  Schedule schedule = guide.getSchedule(channel);

  Program program = schedule.programAt(when);

  return program;
}

Désavantage

Un paramètre en plus : c'est plus complexe

Avantages

En fait, ce n'est pas plus complexe (au contraire)

C'est beaucoup beaucoup plus facile à tester

Il est plus facile de raisonner avec cette fonction

Quelques exemples d'effets de bord

  • Modifier le contenu d'une structure de données
  • Changer l'état d'un objet (setter)
  • Retourner une exception
  • Afficher quelque chose
  • Lire ou écrire un fichier
  • …

Finalement, qu'est-ce qu'une fonction pure ?

On peut dire qu'une fonction est pure lorsque toutes ses entrées sont déclarées comme telles et lorsque toutes ses sorties le sont aussi.

Comment boucler ?

Java


int i;
for(i = 0; i < myArray.length; i++) {
    myArray[i] = myArray[i] + 1
}

Elm


                        List.map (\x -> x + 1) myArray

Comment accumuler ?

Java


int i;
int acc = 0;
for(i = 0; i < myArray.length; i++) {
    acc = acc + myArray[i]
}

Elm


                        List.foldl (\x y -> x + y) 0 myArray
